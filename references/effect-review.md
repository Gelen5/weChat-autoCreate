# 效果复盘流程

## 概述

发布后，通过微信公众号数据回填，分析文章表现，指导后续管道调整。

## 流程

### Step 1: 数据回填

```bash
python {skill_dir}/scripts/fetch_stats.py --article-id MEDIA_ID
```

更新 `history.yaml` 中的数据：
- `read_count` — 阅读数
- `share_count` — 分享数
- `like_count` — 点赞数
- `read_rate` — 完读率（如有）

### Step 2: 数据分析

对比 `history.yaml` 中的历史数据，分析：

| 指标 | 低于平均 | 高于平均 |
|------|---------|---------|
| 阅读量 | 标题/封面可能不够吸引 | 标题策略值得复用 |
| 分享率 | 内容实用性不足 | 内容增强策略有效 |
| 完读率 | 信息密度或排版有问题 | 写作节奏好 |
| 点赞率 | 缺乏情感共鸣 | 人格选择合适 |

### Step 3: 调整建议

基于数据分析给出管道调整建议：

- **阅读量低** → 调整封面策略 / 标题公式
- **分享率低** → 增加可操作内容 / 内容增强
- **完读率低** → 调整段落长度 / 增加视觉元素
- **点赞率低** → 调整人格 / 增加情感锚点
